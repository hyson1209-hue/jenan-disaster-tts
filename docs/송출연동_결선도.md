# 재난 TTS 송출 연동 — 결선도 & 운용 설계

충주 연주소 송출 본선(아날로그 XLR L/R)에 **페일세이프 오버라이드 절체**를 넣어,
야간 무인에도 재난 음성을 자동 송출한다. 자동화 SW/MCFS는 건드리지 않는다(독립 동작).

---

## ★ 현장 적용안 — 기존 A/DPIC-401(HD B&T) 활용 (권장)

송출소에 이미 설치된 절체기 **HD B&T A/DPIC-401**(도면의 "PIC")을 그대로 쓴다.
사양: 입력 4ch(아날로그/디지털 XLR, +4dBu) → PGM 2ch(XLR), Dual PSU, Bypass,
**Fault(무음/저레벨) 자동 절체**(Low −25~−40dBu / Duration 0~45s / Delay 0~3s).

**원리 — "오디오가 있으면 우선 송출"**: 우리 TTS를 A/DPIC-401의 **빈 입력 중 최우선 채널**에 연결.
```
평상시 : TTS 입력=무음 → PIC는 정규방송 채널 송출
재난시 : 송출PC가 TTS 재생 → TTS 입력에 오디오 → PIC가 최우선=TTS로 자동 절체
종료   : TTS 무음 → PIC가 정규방송으로 자동 복귀
```
→ **외부 절체기·USB 릴레이 불필요**(국내 총판 문제 해소). 송출 PC는 **재생만** 하면 됨.

**자재(축소판)**
- 송출 PC + **프로 오디오 출력 IF(밸런스드 XLR, +4dBu)** → A/DPIC-401 빈 입력
- (선택) A/DPIC-401에 외부 원격 채널선택 접점이 있으면 USB 릴레이로 결정적 제어

**소프트웨어 설정**: `TTS_PLAYER=device`, `TTS_OUT_DEVICE=<IF>`, `TTS_RELAY=sim`(릴레이 미사용).
워치독·as-run 로그는 그대로 유효.

**반드시 확인(HD B&T/송출담당)**
1. 빈 입력 채널 유무(4ch 중 사용 현황)
2. **상위 우선순위 자동복귀(auto-restore)** 지원? ← 핵심. 안 되면 아래 대안.
3. 우리 입력 최우선 설정 + Fault 타이밍 짧게 가능?
4. 외부 원격 채널선택 접점(GPI) 유무 / 무음 임계와 PC 잔류잡음 충돌 여부

**대안(auto-restore 미지원 시)**: A/DPIC-401 1대를 출력단에 추가(같은 모델·국내지원),
Input1=TTS / Input2=정규방송 전용 오버라이드 단 구성. 기존 PIC는 미변경.

## (일반형 참고) 외부 페일세이프 절체기 직렬 삽입
국내 PIC를 못 쓰는 경우의 일반 구성. 아래 결선도는 외부 절체기(예: Angry Audio Failsafe Gadget,
또는 국내 PIC형 절체기) 기준. **A/DPIC-401 적용 시 이 절체기는 생략**되고 우리 음원은 PIC 빈 입력으로 들어간다.

## 신호 결선도

```
 ┌──────────────────────── 송출 PC (상시 ON) ────────────────────────┐
 │  input 폴더 재난TXT 감시 → Supertonic TTS → WAV (web/app.py)        │
 │        │                                          │                 │
 │   (밸런스드 라인출력)                         (USB)                  │
 │  [프로 USB 오디오 IF] ──XLR L/R──┐         [USB 릴레이 보드]         │
 └──────────────────────────────────│──────────────────│──────────────┘
                                     │                  │ 접점(GPI)
   MCFS 출력 ─XLR L/R(Main In)─►┌────┴───────────────┐  │
                                │  Angry Audio       │  │
   송출PC 오디오 ─(Backup In)──►│  Failsafe Gadget   │◄─┘  로직입력
                                │  패시브 릴레이       │     engage→Backup
                                │  전원상실=Main 통과  │     release→Main
                                └─────────┬──────────┘
                                          │ Out XLR L/R
                                          ▼
              (기존 경로) ─► EBS(서울망 우선) ─► OPTIMOD 8200/8600 ─► STL ─► 송신(FM 91.9MHz)
```

평상시: Main(MCFS) 그대로 통과 → 송신.
재난시: 송출 PC가 GPI 닫음 → Backup(우리 TTS)으로 절체 → 송출(N회) → 해제 시 Main 복귀.

## 결선 표 (XLR, 2번=Hot/3번=Cold/1번=GND)

| 케이블 | From | To |
|---|---|---|
| Main In L/R | MCFS 출력 (XLR) | Failsafe Gadget **Main In** |
| Backup In L/R | 송출 PC 오디오 IF 출력 (XLR) | Failsafe Gadget **Backup In** |
| Out L/R | Failsafe Gadget **Out** | 기존 다음단(EBS/OPTIMOD 방향) 입력 |
| GPI | USB 릴레이 보드 접점 | Failsafe Gadget **로직입력(원격절체)** |

> 삽입 위치는 **EBS보다 앞단(MCFS 직후)** — 서울망 긴급방송이 항상 최종 우선권을 갖도록.

## 자재(BOM)
- **Angry Audio Failsafe Gadget** (패시브 페일세이프 + 무음감지 + 로직입력, XLR 어댑터 동봉) — 약 $379
  - 후면 XLR 네이티브가 필요하면 대안: **Broadcast Tools Universal 4.1 MLR»Web(XLR)** + 패시브 바이패스 릴레이
- **프로 USB 오디오 인터페이스**(밸런스드 XLR/TRS 출력) — 약 $100~200
- **USB 릴레이 보드**(접점 1ch) — 수만 원
- XLR 케이블 6본(Main 2, Backup 2, Out 2)

## 소프트웨어 (web/broadcast.py)
- `RelayController`: `engage()`(→Backup) / `release()`(→Main). 백엔드 `sim` | `serial`(USB릴레이).
- `Player`: 서버측 재생(`device`=sounddevice로 방송 출력 IF) | `sim`(길이만큼 대기, 무장비 테스트).
- `AirController`: engage → 재생(N회) → **finally에서 무조건 release**(페일세이프) → as-run 로그.
  - **워치독**: `TTS_MAX_AIR`초 초과 시 강제 종료·복귀. PC/재생 멈춰도 정규방송 보호.
  - 동시 송출 거부, 수동 중지(`/api/air/stop`).
- API: `POST /api/air`(긴급송출) · `POST /api/air/stop` · `GET /api/air/status` · `GET /api/asrun`.
- 대시보드: **📡 긴급 송출**(반복 N회) 버튼, 상단 **송출 중 배너 + 중지**, 상태 1.5초 폴링.

## 설정(환경변수) — 실제 장비 전환
| 변수 | 기본 | 설명 |
|---|---|---|
| `TTS_RELAY` | `sim` | `serial`로 바꾸면 USB 릴레이 사용 |
| `TTS_RELAY_PORT` / `TTS_RELAY_BAUD` | `COM3` / `9600` | 릴레이 보드 시리얼 |
| `TTS_RELAY_ON` / `TTS_RELAY_OFF` | (예시 hex) | 절체 ON/OFF 바이트(보드 사양에 맞게) |
| `TTS_PLAYER` | `sim` | `device`로 바꾸면 사운드카드 재생 |
| `TTS_OUT_DEVICE` | (기본장치) | sounddevice 출력 장치명/번호 |
| `TTS_MAX_AIR` | `120` | 워치독 상한(초) |
